# Cyber-Intelligence 用户操作指南

适用对象：舆情分析师、运营值班人员、产品运营人员。
适用版本：一期产品化集成阶段（2026-07-13）。

## 1. 当前可用范围

新版工作台地址：`http://127.0.0.1:8000/`

| 功能 | 状态 | 可执行操作 |
|---|---|---|
| 总览工作台 | 已开放 | 查看声量、BHI、负面数量、最高风险、待办和数据质量 |
| 采集接入 | 已开放 | 检查平台健康与登录状态、启动采集、请求停止 |
| 数据质检 | 已开放 | 筛选复核队列、单条或批量提交人工结论 |
| 情绪分析 | 已开放 | 查看情绪趋势、ABSA、主要话题和 BHI 趋势 |
| 预警中心 | 已开放 | 查看事件详情并执行服务器允许的状态流转 |
| 诉求管理 | 已开放 | 查看诉求聚合并导出 CSV |
| 报告中心 | 接入中 | 新版入口暂不可用；可通过完整跑批或旧报告页面查看 |
| 监控配置 | 接入中 | 新版入口暂不可用；暂使用 `watch.yaml`、`/watch` 和 `/keywords` |

> 页面出现“开发中”并非权限故障，表示对应新版接口尚未开放。

## 2. 启动系统

### 2.1 使用真实业务数据库

首次使用先安装项目并执行自检：

```bash
pip install -e .
python3 -m yuqing.selfcheck
```

启动工作台：

```bash
python3 -m yuqing.dashboard
```

浏览器打开：

```text
http://127.0.0.1:8000/
```

默认使用项目目录下的 `yuqing.db`。停止服务时，在启动终端按 `Ctrl+C`。

### 2.2 使用联调演示数据

联调模式不连接真实平台，也不会覆盖 `yuqing.db`：

```bash
python3 populate_demo_data.py
python3 -c "from yuqing.dashboard import serve; serve(db='yuqing-demo.db')"
```

演示库包含：

- 与当前 `watch.yaml` 对齐的监控对象；
- 正面、负面、低置信、高风险和反讽样本；
- 待复核队列；
- 平台运行记录；
- 一条待确认 P0 事件；
- 一份确定性报告。

如果 `yuqing-demo.db` 已存在，脚本会拒绝覆盖。确认需要重建时执行：

```bash
python3 populate_demo_data.py --force
```

脚本会拒绝把 `yuqing.db` 作为目标，防止误删真实数据。

## 3. 推荐日常操作流程

建议按以下顺序完成一次值班：

```text
检查总览和平台健康
        ↓
启动采集并等待完成
        ↓
处理数据质检队列
        ↓
查看情绪、方面和话题变化
        ↓
确认或处置 P0/P1 事件
        ↓
导出用户诉求，生成周期报告
```

## 4. 总览工作台

进入系统后默认打开总览。

重点查看：

- **总声量**：当前数据库中属于所选监控对象的内容数量；
- **BHI**：品牌健康指数，用于观察整体舆情状态；
- **负面数量/占比**：负面样本规模，不代表全网绝对市场份额；
- **最高风险**：当前负面内容中的最高风险分；
- **进行中预警**：尚未结束的 P0/P1 事件数量；
- **今日待办**：可直接跳转到质检或预警页面；
- **平台健康条**：展示正常、存疑、失败或未知状态。

### 数据质量提示

- `正常/ok`：平台运行记录完整，数据可按当前口径使用；
- `降级/degraded`：部分平台失败、缺失或能力受限；
- `未知/unknown`：尚无足够运行记录，不能把空数据理解为零风险。

## 5. 采集接入

打开左侧“采集接入”。页面显示：

- 当前主机是否具备采集能力；
- 浏览器桥是否正常；
- 各平台最近一次运行时间与采集量；
- 平台是否需要登录以及当前登录状态；
- 当前是否正在运行采集任务。

### 启动采集

1. 确认页面显示“可以运行”。
2. 确认需要登录的平台已经登录。
3. 点击“启动全量采集”。
4. 页面每约 3 秒自动更新一次当前阶段。
5. 运行结束后，进入质检、分析和预警页面查看结果。

系统同一时间只允许一个采集任务。重复点击不会启动第二个任务。

### 请求停止

点击“请求停止”后，系统会在安全检查点协作停止，不保证立即强制终止当前平台操作。

### 平台健康状态

| 状态 | 含义 | 建议 |
|---|---|---|
| 正常 | 采集成功，数量处于正常波动范围 | 继续后续分析 |
| 存疑 | 数量异常、登录可能失效或数据明显波动 | 检查登录态和原平台 |
| 失败 | 采集过程报错或中断 | 重新登录、检测环境后重试 |
| 未知 | 尚无运行记录 | 先执行一次采集，不要判断为“无舆情” |

## 6. 数据质检

质检队列主要收录：

- 机器置信度较低的内容；
- 反讽疑似内容；
- 高风险内容；
- 多模型判断不一致的内容。

### 6.1 筛选队列

可以按以下条件筛选：

- 状态：待复核、已通过、已拒绝或全部；
- 平台：微博、知乎、小红书等；
- 置信度：低于 0.5、0.5～0.8、高于 0.8。

列表默认按风险从高到低排列；风险相同时，置信度较低的内容优先。点击“加载更多”继续读取下一页。

### 6.2 单条复核

| 操作 | 使用场景 |
|---|---|
| 通过 | 机器的现有分析结论可以接受 |
| 拒绝 | 机器结论不可接受，但暂不指定具体修正类型 |
| 标反讽 | 文本实际态度与字面表达可能相反 |
| 标水军 | 疑似重复宣传、灌水或非自然传播内容 |
| 标串味 | 内容与当前监控对象无关 |
| 标正面/中性/负面 | 明确记录人工修正后的情绪方向 |

复核结论会追加写入 SQLite，并记录操作人、时间和备注。刷新页面或重启服务后仍然保留。

> 当前人工结论用于质检审计，不会立即覆盖原始机器分析字段或自动重算历史报表。

### 6.3 批量复核

1. 勾选需要处理的内容。
2. 可点击“全选当前列表”。
3. 点击“批量通过”或“批量拒绝”。
4. 批量拒绝时可填写统一原因。

单次最多处理 100 条。提交期间按钮会被禁用，防止重复写入。

### 6.4 快捷键

先点击一张内容卡片使其处于选中状态，再使用：

| 快捷键 | 操作 |
|---|---|
| `1` | 标正面 |
| `2` | 标中性 |
| `3` | 标负面 |
| `i` | 标反讽 |
| `s` | 标水军 |
| `r` | 标串味 |
| `Enter` | 快速通过 |
| `Esc` | 取消选中 |

“原帖”按钮只允许打开 HTTP/HTTPS 链接，并在新窗口中打开。

## 7. 情绪分析

可以切换近 7 天、30 天和 90 天。

页面包括：

- ABSA 方面级口碑；
- 正面数、负面数、净情绪分和负面占比；
- 主要负面话题；
- BHI 趋势；
- 当前分析样本量及可信度说明。

建议重点关注：

- 净情绪由正转负的方面；
- 负面占比持续上升的方面；
- 新出现或快速放量的话题；
- 数据质量为降级时是否存在平台缺失。

## 8. 预警中心

预警按状态分为：待确认、已确认、已升级、已结束。

### 处置步骤

1. 点击事件卡片打开详情。
2. 查看事件等级、来源文档、最近操作人和备注。
3. 只使用页面显示的“允许操作”。
4. 输入处置备注并提交。

可用操作由服务器状态机决定，例如：

```text
待确认 → 确认 / 抑制
已确认 → 升级 / 解决
已升级 → 解决
已解决或已抑制 → 无后续操作
```

如果状态已经被其他操作更新，旧页面提交可能返回“当前状态不能执行该操作”；刷新页面后重新确认即可。

## 9. 诉求管理

诉求页将真实内容聚合为 Bug、投诉或功能诉求。

可以查看：

- 诉求类型；
- 话题；
- 声量；
- 热度；
- 代表文档编号。

点击“导出 CSV”下载当前监控对象的诉求数据。当前阶段只提供聚合和导出，不会自动创建研发工单，也不支持在页面中修改 Roadmap 状态。

## 10. 报告与监控配置的临时操作方式

### 10.1 生成完整报告

运行完整流水线会执行采集、分析、预警和报告生成：

```bash
python3 -m yuqing.run
```

旧报告页面可通过报告链接或以下路径访问：

```text
http://127.0.0.1:8000/legacy
http://127.0.0.1:8000/report?run_id=<报告运行编号>
```

新版“报告中心”入口将在报告列表、详情、来源溯源和手动生成 API 接通后开放。

### 10.2 修改监控对象

当前监控对象、别名、排除词、危机词和平台列表以 `yuqing/watch.yaml` 为准。

也可以使用旧页面：

```text
http://127.0.0.1:8000/watch
http://127.0.0.1:8000/keywords
```

修改前建议备份并检查 YAML 缩进。旧页面保存时会执行结构校验并生成 `.bak` 备份。

## 11. 常用命令

```bash
# 离线端到端自检
python3 -m yuqing.selfcheck

# 执行一次完整流水线
python3 -m yuqing.run

# 查看 CLI 复核队列
python3 -m yuqing.cli review

# 查看质检统计
python3 -m yuqing.cli review stats

# 生成老板一句话日报
python3 -m yuqing.cli daily

# 舆情问答；无语义服务时自动回退词汇检索
python3 -m yuqing.cli ask "发热问题主要出现在哪些平台"

# 发现新的监控词
python3 -m yuqing.cli suggest
```

## 12. 常见问题

### 页面显示“当前主机未检测到 opencli”

当前主机不具备真实平台采集能力。可以先使用联调数据库体验，或在安装了 opencli 且持有平台登录态的主机上启动服务。

### 平台显示“需登录”

在采集主机上完成对应平台登录。例如：

```bash
opencli xiaohongshu login
opencli douyin login
```

黑猫投诉需要在浏览器中手动登录。登录后回到采集页点击“重新检测”。

### 总览和分析页面没有数据

依次检查：

1. 是否已经执行过采集或使用联调数据库；
2. 数据库中的实体是否与 `watch.yaml` 中的实体 ID 一致；
3. 平台健康是否为未知、存疑或失败；
4. 当前启动命令是否指向预期数据库。

### 写操作返回 403

- 本机访问应使用 `127.0.0.1` 或 `localhost`；
- 远程部署必须先完成飞书 OAuth 登录；
- 反向代理公开域名、协议和 OAuth 回调地址必须保持一致；
- 不要从其他域名跨站调用写接口。

### 联调数据库提示已存在

确认不需要保留原联调结果后执行：

```bash
python3 populate_demo_data.py --force
```

### 无 LLM 或向量 API Key 是否能使用

可以。系统会回退到规则分析、词汇检索和确定性报告；语义召回、模型散文等增强能力可能不可用。

## 13. 使用注意事项

- 当前定位为单分析师、SQLite 单写者场景，不建议多人同时执行批量写操作；
- 不要直接删除或覆盖 `yuqing.db`；操作前应做好数据库备份；
- 平台“零结果”不一定代表没有舆情，应结合健康状态和登录状态判断；
- 重要负面、危机与反讽内容应人工核验原帖；
- 对外输出前应进行来源复核、隐私检查和必要脱敏；
- 采集仅限公开数据，并遵守平台规则及内部合规要求。

- [ ] P0/P1 事件均已确认、抑制、升级或解决；

- [ ] 数据库和重要配置已经备份。
