## 推荐实施顺序

```text
第 1～2 天：0 + 1 + 2
第 3～6 天：3 + 4，前端并行搭建 api/state/view 骨架
第 5～9 天：5，按总览→采集→质检→分析→预警→诉求→报告→配置接入
第 8～10 天：6 + 7
第 11～13 天：8 + 9、真实用户走查和灰度
```

前后端并行边界：任务 0.4 的 API 契约确认后才能并行；在此之前只允许搭建工程骨架，不允许各自定义字段。
