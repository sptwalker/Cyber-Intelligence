# 一期产品化集成提案

## 背景

项目已经具备可复用的采集、清洗、分析、预警、报告、OAuth、SQLite 存储和部署能力，核心自检全部通过；但当前新版工作台仍是 `docs/ui/` 下的静态原型，数据、审批、配置和知识库操作均在浏览器内模拟，未进入 Python 包和 Docker 镜像。

一期目标不是重写技术栈，而是把已经跑通的后端能力接入一个分析师可日常使用的真实工作台，形成“采集 → 质检 → 分析 → 预警 → 报告”的最小闭环。

## 目标

1. 将新版工作台纳入 `yuqing` Python 包、默认 `/` 路由、兼容 `/v2` 路由和 Docker 构建链路。
2. 建立稳定、带版本号的产品 JSON API，替换前端全部内置业务数据。
3. 交付八个真实可用的一期页面：
   - 总览工作台
   - 采集接入
   - 数据质检
   - 情绪分析
   - 预警中心
   - 诉求管理（只读聚合与导出）
   - 报告中心（生成、查看、溯源）
   - 监控配置
4. 保留现有 SQLite、OAuth、ThreadingHTTPServer、Docker 和 K8s 架构。
5. 将单元测试、端到端自检、架构回归和调度自检纳入 CI 阻断项。

## Capabilities

- `workbench-delivery`：将新版工作台纳入 Python 包、默认 `/` 与兼容 `/v2` 路由、wheel 和 Docker 镜像。
- `product-api`：提供统一、版本化、可鉴权的 `/api/v1` 产品接口。
- `analyst-workflows`：接通采集、质检、分析、预警、诉求、报告和监控配置的真实业务闭环。
- `delivery-quality`：建立异常状态、安全渲染、自动化测试、CI 和可回退的直接发布标准。

## 一期范围

### 包含

- 飞书 OAuth 登录和现有 CSRF/来源校验。
- 实体选择、时间范围、加载状态、空状态、降级说明和错误反馈。
- 总览真实指标、BHI、情感趋势、最高风险事件和待办入口。
- 平台采集健康、登录状态、单平台/全平台跑批触发与进度。
- 复核队列筛选、单条及批量通过/拒绝/标脏。
- ABSA、话题、情绪与 BHI 分析数据。
- incidents 列表、详情和合法状态流转。
- 诉求 backlog 实时聚合、CSV 导出；不做研发工单同步。
- 报告列表、报告详情、来源引用和手动生成。
- watch.yaml、关键词、种子建议和平台调度配置。
- 兼容旧页面及现有 API，避免一次性切换风险。

### 不包含

- 结构化知识库及知识编辑工作流。
- 报告草稿—总监审核—发布的多人审批状态机。
- 远程编辑 API Key、Webhook 密钥等敏感配置；继续使用受保护的 `/config`。
- Vue/React SPA、FastAPI、Postgres、Redis、消息队列、微服务。
- 多角色 RBAC、部门数据隔离和多租户。
- 第三方舆情数据采购与新平台适配器。

当前原型中的“舆情知识库”和“系统集成”菜单在一期隐藏或指向说明页，不以模拟功能冒充真实能力。

## 成功标准

- `/` 和 `/v2` 在本地、Docker 和线上环境均能加载新版工作台，其中 `/` 是默认入口。
- 一期页面不再包含 `REVIEW_QUEUE`、`ALERTS`、`REPORTS` 等静态业务数组。
- 所有写操作刷新后仍然存在，并能在 SQLite 中审计。
- 每个页面都有加载、空数据、请求失败和无权限状态。
- 现有 OAuth/CSRF 测试通过，新增 API 契约测试通过。
- CI 自动执行：`unittest`、`selfcheck`、`architecture_check`、`scheduler selftest`。
- 无 API Key 时仍可使用规则分析和确定性报告。
- 单分析师场景下，页面常规查询 P95 小于 1 秒；主动跑批除外。

## 风险与控制

| 风险 | 控制方式 |
|---|---|
| 静态原型功能超过后端真实能力 | 一期导航只开放已定义真实数据源的页面 |
| 新旧 API 同时存在造成回归 | 新接口使用 `/api/v1`，旧接口保留一版兼容期 |
| SQLite 并发写冲突 | 保持单写者、WAL、busy_timeout；跑批和批量标注串行化 |
| 线上容器无法执行 opencli 采集 | 工作台明确显示采集执行环境；一期不宣称 K8s 容器具备浏览器采集能力 |
| 报表数字和页面口径不一致 | 所有读模型复用 `analytics`、`insights`、`report`，禁止前端自行计算业务指标 |

## 实施资源与周期

- 1 名前端 + 1 名后端：预计 2 周完成开发，另预留 3～5 天验收和上线。
- 1 名全栈：预计 3～4 周。
- 需要 1 名实际舆情分析师参与至少两轮页面走查和验收。
