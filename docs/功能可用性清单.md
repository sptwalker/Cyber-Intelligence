# 功能可用性清单

核验日期：2026-07-20
核验范围：当前代码、配置、自动化测试、独立联调数据库和本地工作台 HTTP 冒烟。
结论口径：**代码存在不等于生产可用**。本清单把“已经接通并可验证”“依赖外部条件”“尚待接入或验收”分开记录。

## 结论摘要

- 一期新版工作台的八个页面已经接入真实 `/api/v1/*` 数据接口，不再依赖前端模拟数组。
- 规则分析、风险评分、ABSA、BHI、预警状态机、诉求聚合/CSV、确定性报告和 SQLite 持久化可以离线运行。
- 真实平台采集链路已经有适配器，但必须运行在安装了 `opencli` 或 Collector sidecar、并维护平台登录态的环境中；当前开发工作区没有 `opencli`，也没有启用 Collector，因此不能把“当前环境可采集”写成已验证事实。
- Claude/DeepSeek/MiniMax、Embedding、飞书推送和远程 OAuth 都是条件能力：代码已接入，是否可用取决于密钥、网络、回调和配额配置。
- 多用户/RBAC/Postgres、知识库、审批流、研发工单同步、第三方舆情采购和电商评论 API 不在一期实现范围内。

| 采集接入页面 | ✅ 页面/API 已验证；采集执行条件可用 | `/api/v1/collection/status`、`run`、`stop`、登录状态和运行轮询已接通；没有 `opencli`/Collector 时只显示不可运行，不能抓真实平台。 |

| 情绪分析 | ✅ 已验证 | ABSA、情绪趋势、话题、BHI 趋势和可信度说明复用后端分析模型；无 LLM 时走规则引擎。 |

## 依赖外部条件后可用

### 平台采集

当前 `yuqing/watch.yaml` 默认启用 10 个平台：微博、知乎、小红书、抖音、B 站、贴吧、虎扑、值得买、公众号、黑猫投诉。

| 采集路径 | 代码状态 | 运行前提 | 已知限制 |
|---|---|---|---|
| `weibo`、`zhihu`、`xiaohongshu`、`douyin`、`bilibili`、`tieba`、`hupu`、`smzdm`、`weixin` | ✅ 已有 `opencli`/Collector 调用路径 | 安装 `opencli` 或部署 Collector sidecar；需要的平台完成登录 | 零结果必须结合健康状态判断；登录失效会标记失败而不是“无舆情”。 |
| `heimao` | ✅ 已有浏览器桥解析路径 | 浏览器桥、`YUQING_OPENCLI_SESSION` 和黑猫登录态 | 登录墙、覆盖和稳定性仍需生产环境验证。 |
| 快手 | ❌ 未接入 | 无 | 当前没有 `opencli` 适配器或 Collector 合同。 |

“适配器已实现”只代表代码路径和测试存在；本次没有进行真实平台抓取，也没有验证某个平台当前账号仍处于登录状态。

### 模型与语义增强

| 能力 | 启用条件 | 没有条件时的行为 |
|---|---|---|
| DeepSeek / MiniMax 抽取与分类 | 配置至少一家对应 API Key、Base URL/模型并满足预算；两家同时配置时对重点样本交叉复核 | 降级为规则抽取/分类，不阻塞跑批。 |
| Claude 深度抽取和报告散文 | 配置 `ANTHROPIC_API_KEY` 并安装可选依赖 | 使用确定性摘要和规则结果。 |
| Embedding 语义检索、话题聚类、近似去重、`cli suggest` | 配置 `EMBED_API_KEY`（及可选 Base URL） | 回退到词汇检索；`suggest` 可能提示无建议。 |
| 飞书卡片/日报/告警推送 | 配置 `FEISHU_WEBHOOK` 或飞书应用凭据 | 结果落 SQLite/本地页面，不进行远程推送。 |
| 远程工作台 OAuth | 配置 `FEISHU_APP_ID`、`FEISHU_APP_SECRET`、回调地址和公开 Origin | 本机回环访问可用；远程写接口需要登录，否则 401/403。 |

## 尚待接入或生产验收

这些项目不是当前一期工作台的“隐藏页面”，而是明确的后续工作：

1. **生产部署闭环**：在部署副本初始化 schema、配置 OAuth、持久卷、数据库备份、回滚镜像，并完成线上 `/`、`/v2`、OAuth 回跳、读写接口验收。
2. **真实分析师验收**：由实际分析师完成一轮真实采集 → 质检 → 预警处置 → 报告查看并签字；当前自动化测试不能替代这一步。
3. **无人值守运营**：systemd/NSSM/cron 托管 scheduler、外部 deadman 探活、账号池维护、登录失效响应和 RACI owner 仍需部署与组织接入。
4. **第三方舆情数据源**：识微/清博/鹰眼等采购 POC、字段映射和覆盖/时效验证尚未接入；这是补齐互动量、黑猫和全网覆盖的外部项目。
5. **电商评论 API**：淘宝/京东等自有店铺评论的合规 API 尚未接入。
6. **组织级能力**：Postgres、多用户并发、RBAC、部门看板、多租户和数据隔离未实现。
7. **流程型能力**：知识库编辑、报告草稿—审核—发布审批流、远程密钥编辑、研发工单/Roadmap 自动同步未实现。
